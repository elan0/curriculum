■ 開発
:ウォーターフォール開発　
	開発手順をまとめて1つずつ順番に工程を進める
	企画、設計、実装、テストを一つで凡て実行
:アジャイル開発
	最低限のものから順番に開発していくもの
	各機能の機能一つ一つにそれぞれ細かく工程分けして開発を行っていく
マイグレーション
	システムの完全入れ替え
	他の環境へシステム移行をする事
	exc.windows7からwindows10への移行
:デグレーション
	デグレ、改悪、品質劣化。想定外の不具合が起きたこと
:デプロイ
	ユーザーが使える状態にする作業
	リリースともいう、具体的には実行ファイルを実際に使う場所に配置する事
	exc2.カウントディレクトリの中に配置されたファイルをルートディレクトリ側に配置すること
:コンフリクト
	同じ資源(データ)を使用して動作が衝突する事、
	exc.AさんとBさんが違う作業をしてまとめようとするときに起きがち、
:環境
	機器やソフトウェア、システムを開発するために必要な機材やソフトウェアなどの組み合わせ。
	また、それらを組み合わせて構成された、開発者の作業環境。
:開発環境（ローカル）
	作る側がテストを行う環境
:検証環境（ステージング）
	ユーザーが使う状態に近い環境で作ること（疑似的本番環境)
:本番環境
	ユーザーに実際に利用してもらいながらバグチェック
	出てきた不具合を修正しながら先へ進めていく
:設計書／仕様書
	開発内容の図面/開発の流れ
	どんなプログラムを作れば実現できるか/クライアントの希望する情報
:要件定義
	クライアントの要望をまとめたもの
:基本設計書
	大まかな全体の設計書
:詳細設計書
	具体的な開発内容が書かれたもの
:テスト
	エラーがあるかを検証するもの
	出来るだけ多くのテスト種類を作ってエラーを吐くかを検証していく
:単体テスト（UT）
	各部品ごとに動作確認
	
:結合テスト（CT／IT）
	複数の部品を繋げて行うテスト
:総合テスト（ST）
	最終的な状態でのテスト
:エビデンス／証跡
	テストの結果を記したもの
:データパッチ
	ソフトに変更を加えるための後付けプログラム
:Access
	Microsoftのツールの一つ
	データの加、分析が出来る
:マクロ
	Excelのプログラム
:VBA
	Excelのプログラム
:GAS（Google Apps Script）
	Googleが出してる言語
:API
	言語用の外部機能
:WEB API
	呼ばれるシステム、もしくはその間の
	機能
:REST API
	URLですべてのリソースを一意に示すもの、例えば好きなデータを送るとREST API部分でどの種類のデータで
	どう処理すればいいかを選択してくれる。
:SOAP
	異なるコンピューターで動作するプログラム
	がネットワークで連係動作をするための規約
:Ajax(Asynchronous JavaScript and XML)
	一つのhttpでデータの送受信を完了させられるシステム
	画像の遷移のない通信の事、データを送る、処理する、戻ってくる処理を
	Ajaxで行うことが出来る。その間もユーザーは他の処理を行うことが出来る
:XML
	マークアップ言語の一つ、htmlのようなデータの保存方法
	どんな文字でも入るのが特徴
:JSON
	JavaScript用のデータ交換フォーマット
	XMLとは違うが見やすいデータ
:ステータスコード
	httpの結果が返ってくるコード(404、500、401)
:クロスオリジン（Cross-Origin）／CORS（Cross-Origin Resource Sharing）
	コーズと呼ばれている
	JavaScriptで送受信時のデーの制限を解除するもの
:バリデーション
	記述内容が要件を満たしているかの確認
:データ連携
	格納方法が違うフォーマットを変換してつなげる
:Daemon
	Unix系のOSでバックグラウンドで動くプログラム、もしくは常駐プログラム、
	情報が渡されるまでは待機状態で存在している。
	慣例として最後に[d]をつけるのが普通らしい
:CRON(Linux)
	定期的に起動するOSプログラムの一種、定期的にバックアップを取る、メールの提示配信等に使われる

:バッチ
	複数のプログラムを一つにまとめたもの、もしくはひとまとまりの処理
:ログ
	通信、出力することでどのプログラムを起動したかを確認することが出来る
:CSV
	ファイル形式、カンマ区切りでデータが入力される
	(name,100,20,sword,London)
:TSV
	表形式で保存するデータ形式
:プルリクエスト（マージリクエスト）
	コードの変更を伝える機能
	変更点を見ながらエラーを探せる
